# Python `main` Function Convention

---

## ğŸ”¹ Why Do We Need a `main()` in Python?

- In **COBOL/Java/C#**: Programs have a fixed entry point (`PROGRAM-ID` in COBOL, `public static void main()` in Java/C#).  
- In **Python**: There is **no required main method** â€” the interpreter executes the file from top to bottom.  
- However, using a `main()` function is a **best practice** because:
  - Makes the entry point clear.  
  - Avoids accidental code execution when the file is imported.  
  - Improves readability and modularity.  

---

## ğŸ”¹ The `__name__` Variable

- Python sets a special variable `__name__` in every module.  
- If the file is run directly â†’ `__name__ == "__main__"`.  
- If the file is imported â†’ `__name__` is set to the module name.  

---

## ğŸ”¹ Example Without `main`
```python
print("This runs immediately")

def greet():
    print("Hello from greet!")

greet()
```

- If this file is imported into another file, **all top-level code runs immediately**.  
- This is usually not desirable.  

---

## ğŸ”¹ Example With `main()`

```python
def main():
    print("This is the main function")

def greet():
    print("Hello from greet!")

if __name__ == "__main__":
    main()
```

- When run directly â†’ `This is the main function`.  
- When imported â†’ nothing runs automatically, but `greet()` can still be called.  

---

## ğŸ”¹ Why It Matters

- **Organized Entry Point** â†’ Like Java/C#, easy to find where execution begins.  
- **Reusable Code** â†’ Functions and classes can be imported into other modules without side effects.  
- **Best Practice** â†’ Standard in most production Python projects.

## ğŸ”¹ Real-World Example

### File: `app.py`
```python
def add(a, b):
    return a + b

def main():
    result = add(5, 7)
    print("Result:", result)

if __name__ == "__main__":
    main()
```

### File: `test_app.py`
```python
import app

print(app.add(3, 4))   # Works fine
# app.main() does not run automatically
```
