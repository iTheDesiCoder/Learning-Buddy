## ğŸ”¹ What Do Mutable and Immutable Mean?

- **Mutable** â†’ Object **can be changed** after it is created.  
- **Immutable** â†’ Object **cannot be changed** after it is created; any "modification" creates a **new object**.  

ğŸ‘‰ In Python, **everything is an object**. The distinction is based on mutability, not primitive vs object.

---

## ğŸ”¹ Immutable Data Types

- `int`
- `float`
- `str`
- `tuple`
- `bool`
- `frozenset`

### Example with `int`
```python
x = 10
print(id(x))   # e.g., 140721455673808

x = x + 1      # creates a NEW int object
print(id(x))   # different id
```

### Example with `str`
```python
s = "hello"
print(id(s))

s = s.upper()   # new string created
print(id(s))    # different id
```

ğŸ’¡ **Takeaway**: Immutable objects are safe, predictable, and can be used as **keys in dicts** or **elements in sets**.

---

## ğŸ”¹ Mutable Data Types

- `list`
- `dict`
- `set`
- `bytearray`

### Example with `list`
```python
lst1 = [1, 2, 3]
print(id(lst1))

lst1.append(4)   # modifies the same object
print(id(lst1))  # same id
print(lst1)      # [1, 2, 3, 4]
```

### Example with `dict`
```python
person = {"name": "Alice", "age": 30}
print(id(person))

person["age"] = 31   # in-place modification
print(id(person))    # same id
print(person)        # {'name': 'Alice', 'age': 31}
```

ğŸ’¡ **Takeaway**: Mutable objects allow in-place changes, which can cause **side effects** if multiple variables reference the same object.

## ğŸ”¹ Comparing Mutable vs Immutable

| Feature                | Mutable (list, dict, set)            | Immutable (int, str, tuple)     |
|-------------------------|--------------------------------------|---------------------------------|
| Can modify in place?    | âœ… Yes                              | âŒ No (new object created)       |
| `id()` after change     | Same id                             | Different id                    |
| Safe as dict keys?      | âŒ No                               | âœ… Yes                          |
| Performance             | May be slower if large              | Faster for hash lookups         |

## ğŸ”¹ How It Differs from Java / C#

- **Java/C#**: Distinction is **value types vs reference types**.  
  - Immutable â†’ `String`, wrapper classes (`Integer`, `Double`).  
  - Mutable â†’ `StringBuilder`, collections (`List`, `Dictionary`, `HashMap`).  
- **Python**: Simpler model â†’ everything is an object; the key distinction is **mutable vs immutable**.

## ğŸ”¹ Real-World Example

### Bug from Mutability

### Safer with Immutable
